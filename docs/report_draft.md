**Project:** Walmart Weekly Sales Classification (High vs Low)

## 1) Executive Summary
This project builds a binary classification dataset from Walmart weekly sales by integrating:
1. Internal Walmart Kaggle files (`train.csv`, `stores.csv`, `features.csv`)
2. External macroeconomic data from the FRED API (`UMCSENT`, `RSXFS`, `PCE`)

Current status indicates the data acquisition and integration pipeline is implemented and the validation report is available for the current merged dataset.

## 3) Problem Definition
Goal: classify each weekly store-department sales record as:
- **1 (High):** weekly sales above the store-specific median
- **0 (Low):** weekly sales at or below the store-specific median

This target design avoids bias from store size differences.

### Merge Strategy (Implemented)
1. **Left join** `train` + `stores` on `Store`
2. **Left join** result + `features` on (`Store`, `Date`)
3. **Backward as-of merge** Walmart weekly rows with FRED monthly rows on `Date`
   - uses the most recent previous FRED observation
   - avoids look-ahead bias
